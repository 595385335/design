Java Naming Conventions RFC

Status of this Memo

This document specifies the Java Naming Conventions for the Open Services Gateway Initiative, and requests discussion and suggestions for improvements. Distribution of this document is unlimited.

Abstract

A critical aspect of the Open Services Gateway Initiative specifications for Java is to have a Java Naming Convention that will be followed by all specifications for Java APIs.

Introduction

The OSGi specifications for Java APIs should use the Java package naming conventions in this RFC. The naming conventions specified by this RFC are in general based upon Section 6.8 "Naming Conventions" of The Java Language Specification[1].

Motivation and Rational

It is important to following the naming conventions in this RFC for consistency among API specifications and to avoid namespace collisions with other Java code produced outside of the OSGi development process.

Technical Discussion

Package Names

All Java package names for OSGi specification must be lowercase and must begin with "org.osgi". This will be the OSGi root within the complete Java package namespace. The base package name for the Services Framework will be "org.osgi.framework". The base package names for all service APIs will begin with "org.osgi.service" and be followed by the service name. For example the FooService will use the base package name "org.osgi.service.foo".

Security Considerations

This document has no direct security considerations.  It merely defines the Java Naming Conventions for use by OSGi specifications for the Java programming language..

References
